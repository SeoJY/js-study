Study Summary - #10 Plugin 만들기

객체

  • 강점 : 유지보수에 좋음
  • 단점
    • 다른 개발자와 협업시 호출 오류가 발생할 수 있음.
    • 다른 개발자가 기존에 함수를 선언한 scope와 다른 영역에서 함수를 호출할 경우 가져올 수 없음
      => window 객체를 사용

window 객체

  • 선언 : window.projectName.objName = { } (컨벤션에 따름)
  • 호출 : window.projectName.objName.init();
  • 중간에 projectName도 객체라는 것을 명시해주어야 함 : window.projectName = {};
  • 하이브랩 JS컨벤션 참고

plugin

  • 동일한 UIO가 여러 개 있을 경우에도 동작해야 함.
  • 생성자함수
    • 객체가 아닌 함수로 선언
    • 호출시 인자를 요소로 넣어서 호출
    • var select1 = new 함수명(인자:this가 되어야하는 요소);
  • 호출도 for문으로 plugin화
    • 선언은 함수로, 객체는 prototype에 저장
    • *prototype : 기능들의 집합체. prototype도 객체
    • 호출시 인자를 요소로 넣어서 호출 : var select1 = new 함수명(인자:this가 되어야하는 요소);
    • new로 함수를 호출하면 함수가 객체화 됨 -> 그 내부 this는 함수를 바라봄

    • prop : boolean (보다 정확하고 안정적)
